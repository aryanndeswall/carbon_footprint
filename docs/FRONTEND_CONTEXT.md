# FRONTEND CONTEXT

Project Name: Carbon Sense

Purpose:
Carbon Sense is a behavior-change platform focused on helping users reduce their carbon footprint through activity tracking, daily missions, streak systems, and AI coaching.

Core Principle:
The product is NOT a carbon calculator.

The product IS a sustainability habit-building platform.

---

Primary Features

1. Carbon Dashboard
2. Activity Logging
3. Daily Missions
4. Streak System
5. AI Coach
6. User Profile

---

Technology Stack

Frontend:

* React Native
* Expo SDK
* TypeScript
* Expo Router
* TanStack Query
* Zustand
* MMKV
* Reanimated

Backend:

* FastAPI
* PostgreSQL
* Supabase Auth

AI:

* Gemini 2.5 Flash

---

Design Philosophy

Duolingo + Apple Fitness + Modern Fintech

Keywords:

* Clean
* Motivating
* Premium
* Minimal
* Habit Forming
* Mobile First

---

Primary User Goal

Open app daily.

Complete mission.

Maintain streak.

Reduce emissions.

Receive AI guidance.

Return tomorrow.

---

Important UX Rules

1. Never overwhelm the user with carbon statistics.
2. Show action before analytics.
3. Mission completion should feel rewarding.
4. Streaks should always be visible.
5. Dashboard should focus on today's progress.
6. AI Coach should feel trustworthy and professional.
7. Sustainability Score should be more prominent than raw carbon values in future versions.

---

Current Development Stage

Backend architecture complete through Sprint 5.

Frontend implementation starting.

Frontend should consume existing API contracts.

Do not redesign backend architecture.

Focus on UI, UX, interaction design, component architecture, navigation, and screen implementation.
